Object Exercise 1: Who Heals The Healer?

A lost medical robot was found trudling around near your encampment. People in your community need it desperately. Unfortunaltey, its software seems to have been hacked and it has a lot of errors.

  1. Using the chrome dev console, diagnose the problems and attempt to fix the robot's software in index.js.
  2. Now it runs, and the greet method works (test it out with a name as the argument), but it won't perform its heal method (try to run it). It seems to be locked to a certain location. Unlock it.
  3. Even when you've fixed eR42, eR42 still sometimes makes mistakes (try the heal method a few times on different bodyparts) - and says that the condition is 'undefined'. Why is that? Fix it.
  4. Currently the eR42 has a 50% chance of healing the patient - that's pretty bad. Adjust it up to 85%. 100% wouldn't be any fun, would it?
  5. Wait a second... if something is diagnosed as 'missing', can eR42 heal it? I don't think so. Fix the performHeal method of the medRobotInstructionSetV14 object by implementing a conditional that automatically returns false if the condition is 'missing'.
